## 🎓 Магистерская диссертация: Система тестирования для анализа и синтеза автоматов

## 📋 О проекте
Исследовательская работа в рамках магистерской диссертации по теме:

**«Модульное тестирование системы анализа и синтеза конечных и расширенных автоматов»**

Проект реализует комплексную систему тестирования для библиотеки работы с автоматами, демонстрирующую современные подходы к обеспечению качества математического ПО.

- **Количественный анализ** эффективности различных подходов к тестированию

### 🛠️ Технические результаты:

### ⚡ Ключевые показатели эффективности:
#### 📈 Абсолютная эффективность тестов:
* Hypothesis тесты в 2.16 раза эффективнее на один тест по покрытию
* Один Hypothesis тест покрывает в среднем 3.80% кода, против 1.76% у unit-теста
* 27% тестов (property-based) дают уникальное покрытие, недостижимое детерминированными тестами

### 🔄 Экономика тестирования:
#### ФОРМУЛА ЭФФЕКТИВНОСТИ:
```commandline
Эффективность_теста = (Покрытие_кода / Количество_тестов) × 100%
```
* Unit-тесты: (95.0% / 54) × 100% = 1.76%/тест
* Hypothesis: (76.0% / 20) × 100% = 3.80%/тест
* Отношение: 3.80 / 1.76 = 2.16× (Hypothesis эффективнее в 2.16 раза)

### 🎯 Уникальные возможности Hypothesis:
- Автоматическая генерация сотен тестовых сценариев из одного теста
- Обнаружение edge cases без явного указания в тестовых данных
- Проверка инвариантов системы на произвольных входных данных
- Property-based подход проверяет свойства, а не конкретные примеры
- Адаптивное обучение - кэширование найденных примеров для ускорения последующих запусков

## 🔍 Ключевые концепции
1. Property-based тестирование (Hypothesis)
```# Вместо проверки конкретных примеров проверяем свойства
@given(automaton_data())
def test_encode_states_always_returns_dict(data):
    """Любой автомат при кодировании должен возвращать словарь"""
    fa = create_fa_from_data(data)
    changed, mapping, _ = fa.encode_states(forced_transform=True)
    assert isinstance(mapping, dict)  # СВОЙСТВО: всегда словарь
```

```commandline
    # со смешанными типами, edge cases и т.д.
```

3. AAA паттерн (Arrange-Act-Assert)
```commandline
def test_method_AAA(self):
    # Arrange - подготовка данных
    data = {"transitions": [(0, "a", 1, "x")]}
    
    # Act - выполнение действия
    result = method_under_test(data)
    
    # Assert - проверка результата
    assert result == expected_value
```
